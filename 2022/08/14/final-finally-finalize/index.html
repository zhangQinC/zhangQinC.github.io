final,finally,finalize的区别

2022-08-14 Java SE Java SE, 面试题 Comments

final finally finalize 的区别

final关键字是一个修饰符，意思为最终，不可变

final 关键字用来修饰类，类的成员，以及变量。

  • final修饰类时，表示该类不可被继承，即不可作为父类

  • final修饰方法时，表示该方法不可被覆盖重写，当有子类继承该类时，不可覆盖父类中被final修饰的方法。

  • final修饰变量时，称这些变量为常量，常量在定义时必须进行初始化，而且值不可更改。

    程序中有一个数据应该是固定不变且会多次使用时，将其定义为常量，以增加阅读性并保证其不会被更改。

  • final修饰引用类型变量时，表示该变量的值（存放的内存地址）不可变，即其引用的对象不能变，不能改变引用其他对象，但是引用的对象的内容是可变的。

    因为引用类型变量实际存放的是一个对象的内存地址，它的值就是一个内存地址，final保证了这个存放的内存地址不可变，即只能固定引用一个初始化时引用的对象。不可更改。但是对象可以改变自己的属性，因为对象一旦创建，内存地址就固定了。

finally关键字在进行异常捕获时使用try/catch/finally中的语句块

在try/catch语句块中，无论是否有异常发生，finally中的语句一定会执行。

例如一些重要的清理工作。

注：

  • finally语句是可选的
  • finally中的代码一定会执行
  • 可以使用try/finally

finalize()是Object类中的方法

finalize()方法会被在垃圾回收前自动执行，不需要主动调用。
